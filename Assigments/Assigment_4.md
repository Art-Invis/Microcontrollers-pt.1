## 💡 Лабораторна робота №4

### Курс: *Мікроконтролери Ч.1*
### Тема: **Реалізація хмарного обміну даними через протокол MQTT (HiveMQ Cloud)**

**Варіант:** Комплексне завдання

---

### 📌 Мета:
Ознайомитися з принципами роботи MQTT-протоколу як одного з ключових механізмів обміну повідомленнями у системах Інтернету речей (IoT). Навчитись підключати мікроконтролер ESP8266 до хмарного MQTT-брокера через захищене TLS-з’єднання, реалізувати прийом команд, їх обробку та відображення результатів на OLED-дисплеї.

---

### 🛠️ Завдання:

1. **Мережева конфігурація:**
   * Підключити мікроконтролер до бездротової мережі Wi-Fi.
   * Налаштувати MQTT-клієнт для встановлення безпечного TLS-з'єднання з хмарним сервером **HiveMQ Cloud**.
2. **Маршрутизація повідомлень (MQTT):**
   * Реалізувати підписку (Subscribe) на топік: `emqx/esp8266`.
3. **Обробка команд (Payload parsing):**
   * При отриманні команди `WATER_CHECK`: ініціювати зчитування актуального рівня води з аналогового датчика.
   * При отриманні будь-якої іншої (невідомої) команди: генерувати обробку помилки.
4. **Індикація та UI:**
   * Вивести результати успішної перевірки рівня води на OLED-дисплей.
   * Вивести повідомлення "Unknown command" на дисплей у разі помилки.
   * Дублювати всі статуси підключення та отримані payload-дані у Serial Monitor для відлагодження.

---

### ⚙️ Використані компоненти та протоколи:

| Технологія / Компонент | Призначення |
| :--- | :--- |
| **MQTT** | Легковаговий протокол обміну повідомленнями (Publish/Subscribe) |
| **HiveMQ Cloud** | Віддалений хмарний брокер |
| **TLS/SSL** | Криптографічний протокол для безпечного з'єднання |
| **OLED 0.96" (I²C)** | Локальний графічний інтерфейс |
| **Analog Water Sensor** | Сенсор рівня рідини (ADC) |

---

### ✅ Результат:
Створено захищений MQTT-клієнт на базі ESP8266. Пристрій успішно приймає команди з глобальної мережі через брокер HiveMQ, коректно їх парсить, взаємодіє з фізичними сенсорами та забезпечує миттєвий зворотний зв'язок через локальний дисплей і монітор порту.
